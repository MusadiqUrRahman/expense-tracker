# ⚡ Session — Claude Code Concept

## 📌 Purpose
A session in Claude Code is one full conversation from start to end.

---

## ⚙️ What It Does
- Starts when you run `claude`  
- Ends when you use `/exit`  
- Saves full message history  
- Tracks all actions (file reads, tool results, commands)  
- Keeps current working directory context  

---

## 📂 Session Storage
- Sessions are saved automatically  
- Stored inside:
~/.claude/projects/

- You can resume sessions anytime  

---

## 🚀 How It Works
- One session = one complete workflow  
- Claude remembers everything inside the session  
- Helps you work step-by-step without losing context  

---

## 🔄 Session Features
- Unique session ID  
- Full conversation memory  
- Resume previous work  
- Context-aware coding  

---

## 💡 When to Use
- For one complete task or project  
- When you want Claude to remember your work  
- When working step-by-step  

---

## 🧠 Pro Tip
Use a new session for each task to keep things clean and accurate.  

---

## ⚠️ Notes
- Long sessions can become heavy  
- Too much context may reduce accuracy  
- You can exit anytime using `/exit`  

---

## 📚 Summary
Session = One full working conversation  

---
