# ⚡ /resume vs claude -r — Claude Code Commands

## 📌 Purpose
Both `/resume` and `claude -r` are used to **continue a previous session**, but they work in different ways.

---

## ⚙️ Key Difference

### 🔹 /resume
- Used **inside Claude (interactive mode)**  
- Resumes the **last session automatically**  
- No need to enter session ID  

### 🔹 claude -r
- Used **in terminal (outside Claude)**  
- Requires a **session ID**  
- Lets you resume a **specific session**  

---

## 🚀 Usage

### ✅ /resume (inside Claude)
/resume

👉 Simple: continues your last conversation  

---

### ✅ claude -r (in terminal)
claude -r "session-id"

Example:
claude -r "abc123"

👉 Advanced: resumes a specific session  

---

## 💡 When to Use

### Use `/resume`
- When you just want your **last session back**  
- When working quickly  
- When you don’t care about session ID  

---

### Use `claude -r`
- When you want a **specific old session**  
- When working on multiple projects  
- When restarting from terminal  

---

## 🧠 Pro Tip
Use `/resume` for quick work, and `claude -r` for **controlled and precise session recovery**.

---

## ⚠️ Notes
- `/resume` = automatic (last session)  
- `claude -r` = manual (needs session ID)  
- Both keep your previous context  

---

## 📚 Summary
`/resume` = Quick resume  
`claude -r` = Resume with control
