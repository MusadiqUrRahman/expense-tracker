# ⚡ /init — Claude Code Command

## ⚠️ Important Note
- `/init` creates a **CLAUDE.md file automatically**  
- It scans your project files to understand your codebase  
- The generated file is only a **starting point — you must review and edit it**  
- More detailed and clear instructions in CLAUDE.md = better results from Claude  

---

## 📌 Purpose
`/init` is used to **initialize a CLAUDE.md file that documents your project and guides Claude Code behavior**.

---

## ⚙️ What It Does
- Creates a new `CLAUDE.md` file  
- Scans your project directory  
- Understands structure, files, and technologies  
- Writes initial documentation automatically  
- Sets rules and context for Claude  

---

## 🚀 Usage
/init

---

## 🧠 How It Works
1. You run `/init`  
2. Claude scans your project files  
3. It analyzes code, structure, and tools  
4. It generates a `CLAUDE.md` file  
5. This file is used in future sessions  

👉 Claude reads this file every time you work in that project :contentReference[oaicite:0]{index=0}  

---

## 📂 What is CLAUDE.md?
`CLAUDE.md` is a **project instruction file** that tells Claude:

- How your project works  
- Coding rules and style  
- APIs and structure  
- Development workflow  
- Any custom instructions  

👉 It acts like a **brain or guide for your project**  

---

## 💡 When to Use
- When starting a new project  
- When setting up Claude for a codebase  
- When you want consistent results  
- When working in teams  

---

## 🧠 Pro Tip
After running `/init`, always:
- Open `CLAUDE.md`  
- Remove unnecessary parts  
- Add your own rules and instructions  

---

## ⚠️ Notes
- Auto-generated content may not be perfect  
- You must customize it for best results  
- File is a “living document” (can be updated anytime)  

---

## 📚 Summary
`/init` = Create project brain (CLAUDE.md)
